Set up the basics

  1. Initialize NPM in Blazor

    Add a folder called NpmJs at root in your project to be able to install NPM packages.

  2. Install CSS library and web bundler

    Install esbuild and the CSS library to get the basic components and styling.

    The packages acts like any other external dependency you install and update in your project.

  3. Create index.js

    Create a new file index.js and import the CSS library files.

  4. Add CSS-resetOptional

    Import the Elvis CSS reset if you wish to apply a generic CSS reset to your application.

    Be aware that you may break styles in your app if you import this in an existing application, since you may rely on browser defaults without knowing it. This is why this import is separate from the required Elvis import.

  5. Build-script setup

    Create a script in package.json to build the file you created.

  6. Run the build script

    Run the new build script that was created.

  7. Import built files

    Import the stylesheet and script that was built. The js/index.bundle.js script should be added after the framework/blazor.web.js script.

  8. Start using classes

    Choose between multiple brand utilities and components. The components have multiple variants and options to fit different needs.

Add an advanced component

  1. Install component

    Install the datepicker component.

    The packages acts like any other external dependency you install and update in your project.

  2. Add imports

    Import the component.

  3. Start using advanced components

    Choose between several different advanced component and customize according to your needs by using the available props. The datepicker API lists all the properties and events available for datepicker.

    Updating props and events

    Blazor requires a bit more setup for events to work, read about using NPM packages in Blazor . Remeber Blazor uses native JavaScript, which means for updating props and listening to event you have to use addEventListener.
