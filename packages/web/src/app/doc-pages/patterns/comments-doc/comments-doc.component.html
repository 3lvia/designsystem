We use comments for leaving notes, or to help users communicate with each other. Users can see others’ comments and write, edit and delete their own. These actions are visible to users when hovering over their comments.

The comment input field should be easy to find and always visible, and it should therefore be sticky. The comment field is flexible in width and height but has a minimum width of 325px and a maximum of 600px. You can use both normal and compact size of the input field.

When there’s no comments in a comment area, leave an element to show where the comments will appear. You can add an illustration together with text if suitable. See more in the empty states pattern.
