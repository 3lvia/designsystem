Bar charts are graphs that show categorical data by bars proportional to the value they represent. They are best used to compare groups or to track large changes over time. When using, make sure to scale the y axis so that all data can fit in the graph.

Simple bar charts are for categories with clear data set and often only requires one data color. The bars can be manipulated with the filter buttons on the top right of the bar chart.

Advanced bar charts include interactivity within the graph. Specific graph data is shown on hover, and can be detailed when clicking.

Pie charts show parts of a whole and are good when displaying data of 6 or fewer categories.

The colors used in graphs are our data colors that have been specially designed to be easier to distinguish from each other, even for people with certain color blindness. They should be used in the order as presented. Data color tints are used when hovering in advanced charts. To make sure you create accessible solutions, remember to never rely on color alone to visualize important data. You can read more about our color guidelines in our colors documentation.

The tooltip component is used to show additional information on hover. This is recommended to be kept short, but depending on data and case, exceptions might have to be made. Legends are used close to the graph and should always be visible.

Graphs should be used on white backgrounds. Avoid any darker color background, since the contrasts will not be kept.

Note that the bars are only slightly rounded (4px). This is to not obscure data and make sure the graphs are easy to read.

Components like date picker, segmented control, button and chip can be used in combination with the graph to filter out data.

We do not provide any code or styling for implementing graphs. Instead, you should follow the guidelines and styling provided with examples here.
