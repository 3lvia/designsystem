Icons are more easily recognizable than text, language independent, aesthetically pleasing, and help separate actions from content. Using a common dictionary of icons across Elvia makes it easier for users to interact with our platform and secure a holistic visual communication and identity. Icons are typically used in combination with text, and only sometimes unaccompanied. Only if a text label does not fit and is not needed to understand the signification. One of the most common UX mistakes is assuming that the users understand the icons we choose. Our icon library is based on Streamline icons. This license includes royalty-free usage in an unlimited number of projects without the need to credit the source. Note that you can only use our icons in Elvia projects, and they may not be distributed outside of the organization. You can read more about Streamline and their license guidelines here .

Icons are available in sizes from XXS to XXL based on multiplications of 8 pixels. When choosing an icon size, keep in mind the screen size and accessibility considerations. Mobiles and tablets require a larger touch surface and disabilities can make it difficult to interact with and/or see icons that are too small.

  • e-icon--xxl - 56px
  • e-icon--xl - 48px
  • e-icon--lg - 40px
  • e-icon--md - 32px
  • e-icon--sm - 24px
  • e-icon--xs - 16px
  • e-icon--xxs - 8px

The green color on icons is typically used for the hover state, indicating that it is interactive. A small pop-of-green is also used to draw focus to the most important detail to better understand the significance of the icon. However, these icons are typically not interactive. The colors red and orange are reserved to convey errors and warnings to the users.

Check out our color palette for a more in-depth understanding of the Elvia colors and how to use them.

Inverted icons should only be used on darker backgrounds or for dark mode.

Grayed-out icons signify that the interaction is disabled and can not be clicked. Try to avoid disabling any elements. Not all users understand the disabled state, confusing it with broken â€“ leading to a frustrating user experience.

Disabled elements often create trouble for users with vision impairments, critiqued for their low contrast and leaving out critical information for users dependent on screen readers. Try using appropriate contextual feedback instead.

The icon fill depends on what background the icon is placed on. Filled icons are typically better for readability on darker backgrounds, while outlines are often sufficient for lighter backgrounds. Be mindful of contrast.

The stroke weight of icons should match thickness of the surrounding typography.

Additionally, itâ€™s important to be consistent in the choice of fill and weight â€“ don't mix outlined icons with filled icons, or use differing stroke weights for grouped icons. Be consistent.

Elvis-assets-icons npm package
All icons are accessible in a separate package from Elvis ( @elvia/elvis-assets-icons ) as SVGs within a JS module. Import the required icon individually where you need them:
  • To get the icon you can use the following:
    addCircle.getIcon() .
  • If you want a inverted icon use the following:
    addCircle.getIcon('inverted') .
  • If you want a color:
    addCircle.getIcon('white') .
  • Or set the color yourself with a hex:
    addCircle.getIcon('#bdbdbd') .
