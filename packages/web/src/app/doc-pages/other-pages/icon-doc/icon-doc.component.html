In using icons the correct way, the benefits are plentiful. Icons are more easily recognizable than text, language independant, aesthetically pleasing, and help separate actions from content. Using a common dictionary of icons across Elvia makes it easier for users interact with our platform and secure a holistic visual communication and identity. Icons are typically used in combination with text, and only sometimes unaccompanied. Only if a text label does not fit and is not needed to understand the signification. One of the most common UX mistakes is assuming that the users understand the icons we choose. Our icon library is based on Streamline icons. This license includes royalty free usage in an unlimited number of projects without the need to credit the source. Note that you can only use our icons in Elvia projects, and they may not be distributed outside of the organization. You can read more about Streamline and their license guidelines here .

Our icons are available in sizes from XXS to XXL based on multiplications of 8 pixels. When choosing an icon size, keep in mind the screen size and accessibility considerations. Mobiles and tablets requires a larger touch surface and disabilities can make it difficult to interact with and/or see icons that are too small.

The green color on icons is typically used for the hover state, indicating that it is interactive. A small pop-of-green is also used draw focus to the most important detail to better understand the significance of the icon. However, these icons are typically not interactive. The colors red and orange is reserved to convey errors and warnings to the users. Check out our defined color palette for a more in-depth understanding of the Elvia colors and how to use them.

Inverted icons should only be used on darker backgrounds or for dark mode.

Gray outed icons signifies to the users that the interaction is disabled and cannot be clicked at this time. However, always try to avoid disabling any elements. Not all users understand the disabled state, confusing it with broken, and lead to a very frustrating user experiance. Moreover, disabled elements often creates trouble for users with vision impearments, critiqued for itâ€™s low contrast and leaving out critical infromation for users dependant on screen readers. Try using appropriate contextual feedback instead.

The icon fill depends on what background the icon is placed on. Filled icons are typically better for readibility on darker backgrounds, while outlines are often sufficient for lighter backrounds. Be mindful of contrast.

The stroke weight of icons should match thickness of the sourrounding typography.

Additionally, itâ€™s important to be consistent in choice of fill and weight - donâ€™t mix outlined icons together with filled icons, or use differing stroke weights for grouped icons. Be consistent.

Elvis-assets-icons npm package
All icons are accessible in a seperate package from Elvis (@elvia/elvis-assets-icons) as SVGs within a JS module. Import the required icon individually where you need them:
  • To get the icon you can use the following:
    addCircle.getIcon() .
  • If you want a inverted icon use the following:
    addCircle.getIcon('inverted') .
  • If you want a color:
    addCircle.getIcon('white') .
  • Or set the color yourself with a hex:
    addCircle.getIcon('#bdbdbd') .
