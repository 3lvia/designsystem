The design system helps designers and developers create software with a consistent look and feel and saves time by allowing us to reuse components and design elements across multiple systems and sketches.
We separate the available components into a style library and separate components. The style library is called elvis and can be used to get styling with html classes. For more advanced components we offer separate packages that can be used individually. When browsing our components it will be clear what kind of component it is by looking at the code examples or the installation guidelines. Find all the components under "Components" in the main menu.
Elvis is the styling library we offer with multiple component classes and utilities like colors, icons and typography.

As a designer, there’s a few resources to gather before getting started with the design system in Elvia. First thing to do is to make sure you have access to the Figma library.
You can use Figma as an online tool, but we recommend you to download the app for the full experience. If you have any issues accessing the Elvia library , just ask someone in the team and we’ll send you an invite. We can of course also help you set up libraries in Figma if you’re new to it.

In our design language, we use round forms and circles in combination with our green profile color to symbolize environmentally friendly energy and to show that Elvia is ON, activated and fully charged. You can read more about the brand on brand.elvia.io.
The main colors we use are black, grey, white and green. The green color specifically should only be used in details and - in the cases where it works - to symbolize that something is on, or active. There’s a bit more tips to help you with color usage in our color guidelines.
The typography we use is available in two optical sizes: Red Hat Text and Red Hat Display. The Display version is most often used in bigger texts, such as titles, while the Text version is used in standard text blocks, to provide better readability. The font family is open-source and Red Hat can be downloaded here via Google Fonts. You can read more about our usage of it on our typography page.
For our icon library, we use Steamline as a base. We do however do find ourselves in cases where we need to tailor icons to fit our specific needs, and adjust when we need to. To see the full overview of available icons in Elvia’s design system, see our icons page. We continuously add new ones depending on demand, so if you can’t find what you need for your project, let us know.
In Elvia, we follow the 8 point grid. If you haven’t worked with it before we recommend you to read an intro to the 8 point grid. Through the 8 point grid we can ensure higher consistency through using multiples of 8 to space out elements. This helps us define dimensions, paddings and margin to make sure that both components and full pages are scalable for a wide variety of devices.
We have defined breakpoints for different types of devices that we follow to make sure we create responsive interfaces. You can read more about this in the breakpoints section.
In Elvia, designers help each other to create great experiences. You can find information about Elvia’s design process and some suggested methods on our pages about our design process, as well as important accessibility pointers to help you make sure that you deliver solutions that are inclusive.
You are welcome to reach out to any of the designers in the design system team if you have questions or ideas about the design or usage of the design system, or you can join our design recaps each week to get some great input from the full design team.

Contentful is a content management system that the design system is using to make it easier for non-developers to edit content on design.elvia.io. Contact us if you think you should have access to edit content on design.elvia.io or you need help with anything.
  • CMS - Content management system, a system that lets people edit content easily.
  • Contentful - The CMS the design system have chosen to use.
  • Preview - A way to view content that has not been published yet.
  • Entry - Entries are different types of models that can be used on a page, e.g. images or links.
  • Models - Models are small code bundles we have made to enable us to add the different parts of a website the way we want it to look like. These models are what you use to add entries, e.g. images or links.
You need an Elvia account to log in to Contentful. If you have received access use SSO in the bottom of the page to sign in.
Next you will be asked to fill in SSO name, this field should be 'Elvia'. After that proceed to sign in a normal.
The contentful structure is made up of different content types. In the content tab you can find all the content that is relevant for you to edit. Every page on design.elvia.io is called a 'documentation page' in Contentful. These documentation pages are the bases for adding or editing content.

Sorting

You can easily sort and search through all the content in Contentful in the 'Content section' (see image above) . You can choose to sort by content type or other sortable features from the dropdown inside the search-field as well as from the sidebar.

Editing options

When editing a documentationPage you will mostly work inside the content field. In this field you can use the Contentful options at the top to make headings, bold text, lists and quotes among other things.

Adding entries

You can also add / embed entries at the top right of the field. Entries are models we have made to make it possible to customize the content more. When adding an image or link you should always add them by using these models. If you want the entry to be inline with the text you should choose "inline entry." Inline images are added in a different way that can be read about below in the "Useful models and how to's".
When you are adding entries you can choose to either use an existing entry and search through what has already been added, but mostly you will be adding new entries. Adding a new entry is done by clicking the "Create new entry" button in the top right corner of the "Add entry" popup.
Some fields are required and will show a red border around it if they are not filled in. Some fields also have help-texts below them to help explain what the specific field does or how it should be filled in.
These are some of the models and options you can use to edit content in Contentful. If you want to see examples of how to do something you can easily sort content by type in the lift sidebar when you are on the "content" tab.
  • Submenu - Submenues are the main menu-items (about, brand, components, patterns, tools). If you are adding new documentation pages you must also add the page to the submenu entry for that menu in Contentful. If you are adding a new brand page, remember to also add a new 'Overview card' to the landing page for that submenu.
  • Documentation page - The documentation pages should serve as the base for all content on a page. These pages can be previewed with all content, even if that content also is just in draft. The page needs a title, path and content and should usually also have a description. When adding a new documentation page you also have to add it to the correct submenu.
  • Image - Images should be added through the image entry model and the image require a name, alt-text and an asset / media. If you want the image to be placed next to text you should put that text in the "Content next to image" and then chose the correct alignment in the "Alignment" field.
  • Links - All links should be created through the link entry model. Links have to have a link-text and either an url to an existing page on design.eliva.io or an url to an external page. When linking to pages on design.elvia.io always use the "Url design.elvia.io (internal)" field.
  • Centered content - You can center any content by using the centered content model in Contentful and then add the content that should be centered there.
  • List - Make lists through the Contentful option-bar at the top of the content field.
  • Icon list - To make a icons list can be made by starting each bullet point with an inline image. The image should an icon and the size is 32px X 32px.
  • Download content - If you want to offer downloadable files or images on design.elvia.io, e.g. illustrations or logos, you can use this model. For images use SVG format preferably. Each download content entry you make will get its own button for downloading.
  • Grid - The grid model can be used to place download content or images in a grid view (multiple next to each other). Make sure to choose either download content or image and not both in the same grid.
  • When & When not to use - The model generates lists of points on when to use and when not to use a something, mostly used for component.

Previews

When testing your content you have the "preview" option available in the right sidebar. This will open a link in a new tab showing how the content will look after you publish it. It will also display error-messages if your content is incorrect. It is therefore important to always check the preview before publishing to make sure the content is correct. The preview button is only clickable on documentation pages, but all content on that page will be visible in the preview, even if it's not published yet.

Publish

Publish only once all the content is fully ready. Most of the time you should be working with your content in draft without publishing it and use the preview until you are happy with the page and are ready to publish. You can easily send preview-links to other people for them to review your changes before publishing, to ensure that everything is correct. You will not be able to publish your content if you have errors, but you should always check the documentation page preview before publishing anyways.
Publishing anything will also trigger a new build and deploy of the design system website. This means that you should only publish once something is completely finished so that the build does not have to be triggered all the time.
You can also publish multiple things at the same time by e.g. sorting on a content type on the "content" tab page and then marking all the checkboxes before pushing the "Publish" or "Republish" button.
Any entry can be deleted as long as you are sure the entry is not used anywhere in Contentful. If you are creating an entry by mistake, you should delete it so that there is not a lot of unused entries floating around. If you want to be sure that you are not deleting an entry that is being used, look at the right sidebar under links.
You can also hover over the link symbol in the top left corner next to the title of your entry to see references.
