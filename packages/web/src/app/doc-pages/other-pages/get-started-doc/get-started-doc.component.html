Font
Elvia Designsystem uses two fonts provided by Google:
  1. Red Hat Text
  2. Red Hat Display
The import of these two font types are provied within the compiled CSS of Elvis, and you don't need any font imports in your project to get the correct font style for Elvia styled components.

Colors and style
The main colors we use are black, grey, white and green. The green color specifically should only be used in details and - in the cases where it works - to symbolize that something is on, or active. Thereâ€™s a bit more tips to help you with color usage in our color guidelines.
The typography we use is available in two optical sizes: Red Hat Text and Red Hat Display. The Display version is most often used in bigger texts, such as titles, while the Text version is used in standard text blocks, to provide better readability. The font family is open-source and Red Hat can be downloaded here via Google Fonts. You can read more about our usage of it on our typography page.

Grid and breakpoints
In Elvia, we follow the 8 point grid. If you havenâ€™t worked with it before we recommend you to read an intro to the 8 point grid. Through the 8 point grid we can ensure higher consistency through using multiples of 8 to space out elements. This helps us define dimensions, paddings and margin to make sure that both components and full pages are scalable for a wide variety of devices.
We have defined breakpoints for different types of devices that we follow to make sure we create responsive interfaces. You can read more about this in the breakpoints section.

Contentful structure
Contentful is structured up of different content types. In the content tab you can find all the content that is relevant for you to edit. Every page on design.elvia.io translates to a 'documentationPage' in Contentful. These documentationPages are the bases for adding or editing content.
