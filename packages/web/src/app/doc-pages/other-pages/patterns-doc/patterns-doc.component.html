A form is a group of input fields and selections for users to enter data. Forms can be simple or complex, but always try to make it easy, clear and quickly for the user to use.

One column

To improve the scanability of a form stack the input fields vertically and display it as a single column. This makes it easier and faster for user to scan. Exceptions for this is related fields like postcode and city or date of birth.

Mark optional

Some input fields can be optional or required. In Elvia, we mark the optional fields. Most times there are more required fields than optional, so to avoid visual noise and use a more positive tone to it, we write “optional” behind the label to indicate who’s optional.

Length of the input field

The size of the input field should reflect the expected answer. For example, a postcode field should therefore be shorter in size than the address field.

Complex forms

Forms can be long and complex and this can put some users off. To make a better experience for the user you can group related questions and break up the form in separate pages or steps. Use a stepper to inform the user how long the form is and where they are in the process.

Keep it simple, so try to remove all non-essential fields. Ask the user for truly necessary information and remove the rest.

Enable the buttons

You often disable a button untill all the required fields and inputs are filled out. A disabled button can make the user irritated, confused, and make the user feel stupid. A better solution is to enable the button and rather guide the user to the missing fields with red error indications when they are trying to submit the form.

Empty states are moments when there is no data and nothing to display to the user. In these cases the user needs to be informed what’s happenening and what to do about it.

Example of empty states
  • Error page, like 404
  • Searching for something but no results was found
  • Confirmation page when you have completed something, like submitting a form
Inform the user

A good empty state should explain what’s going on in plain language. Tell the user what the next step is and guide them forward with a button or link if needed.

Tone of voice

Don’t use product-specific terms that the user may not understand. Use a positive tone and motivate the user, maybe by the use of humor or add some emojis in the message.

Visuals

Use a graphical elements to support the information such as illustrations, colors and icons. You can use a neautral or humorous tone to it, but try to play with our concept off/on - most often this is a great opportunity for when something is off!
