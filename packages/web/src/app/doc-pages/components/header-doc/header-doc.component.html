The internal header is used on all internal applications in Elvia. This page documents how to use and implement this header. The external header should only be used on applications when they are meant for external users, users outside Elvia. The external version of the header has dark background instead of light and usually doesn't require any login or profile. If you need login with your external header contact us and we will consider offering the external header as a component.
The header contains a general top menu and a side navigation section. The navigation bar on the side contain app-specific tabs, while the top menu gathers general navigation, such as the app switch and any global settings. Since routing usually is performed by a framework-specific library, the DOM for the sidenav items is written in your app. This enables use of routing-library features, such as setting active sidenav item and changing icon between colored and grayscale. See the examples in the CEG on how the DOM structure should be like.
Use the CSS classes
  • e-sidenav__container
  • e-sidenav__icon-container
  • e-sidenav__item
  • e-sidenav__item-text
 The active state of sidenav items can be toggled with the CSS class e-sidenav__item--active and should be added to the link element that already has the class e-sidenav__item . The toggle of the active class depends on your framework or routing library, but most modern routing libraries have an attribute for setting an active class on routes (see for example routerLinkActive for Angular or activeClass for Vue).

Use a colored icon from our icon library for active sidenav items. If your desired icon does not have a colored variant, please contact Team Atom .

You can include additional content besides the page title in the Header by using the pageTitle as a slot rather than just a string. For instance, you can add an icon button or a status message. Sometimes you may need to know the current theme. A function getCurrentTheme is therefore exposed by the header package. Simply import it from @elvia/elvis-header (or @elvia/elvis-header/react for React) and call it to get either 'light' or 'dark' in return.
