NB! This preview is not a finished working header.
This header only shows some logic to visualize how it should look. Logic has to be applied for opening, closing and choosing menus and items.
  • The modifiers classes listed below should be added and removed when clicking menus and choosing options.
  • When clicking outside of any menu or if choosing any option the menus should close.
The internal header contains a general top menu and a side navigation section. The navigation bar on the side contain app-specific tabs, while the top menu gathers general navigation, such as the app switch and any global settings. When using icons in the sidebar of the header, you will need both a black-and-white and a two-colored icon, for the active state. If you are using icons from the design system and a colored version of the icons you have chosen does not exist, contact us and we will make it for you. To open a menu in the top-bar one or more of the following classes needs to be added.
New class Where When
e-header__top-bar-desktop__app--open e-header__top-bar-desktop When opening app-menu
e-header__top-bar-desktop__area--open e-header__top-bar-desktop When opening area-menu
e-header__top-bar-desktop__user--open e-header__top-bar-desktop When opening user-menu
e-header__sidebar--open e-header__sidebar When opening side-menu
e-header--open e-header When opening top-menu on mobile
e-header__top-bar-mobile__app--open e-header__top-bar-mobile When opening app-menu on mobile
e-header__top-bar-mobile__area--open e-header__top-bar-mobile When opening area-menu on mobile
e-header__top-bar-desktop__app-circle--active e-header__top-bar-desktop__app-circle Should always be present on 1 element
e-header__sidebar__item--active e-header__sidebar__item Should always be present on 1 element
