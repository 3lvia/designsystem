Header {{ description }}
The header contains a general top menu and a side navigation section. The navigation bar on the side contain app-specific tabs, while the top menu gathers general navigation, such as the app switch and any global settings. Since routing usually is performed by a framework-specific library, the DOM for the sidenav items is written in your app. This enables use of routing-library features, such as setting active sidenav item and changing icon between colored and grayscale if desired. See the examples in the CEG on how the DOM structure should be like. The active state of sidenav items can be toggled with the CSS class e-sidenav__item--active and should be added to the link element that already has the class e-sidenav__item . The toggle of the active class depends on your framework or routing library, but most modern routing libraries has an attribute for setting an active class on routes (see for example routerLinkActive for Angular or activeClass for Vue).
