A good empty state should explain what's going on in plain language. Tell the user what the next step is and guide them forward with a button or link if needed.

Don't use product-specific terms that the user may not understand. Use a positive tone, motivate the user and add an emoji in the message to express personality.

No result (search)

  • "Her var det ingen treff. Prøv med et annet søk."
  • "Ingen resultater. Prøv et annet søk eller fjern noen filtre."

No data (first use, cleared by user)

  • "Her var det tomt. Kom i gang ved å legge til noe."
  • "Ingen ting å se her. Kom i gang ved å legge til noe."

Error

  • "Noe gikk galt. Prøv igjen senere."
  • "Vi klarte ikke å hente data. Prøv igjen senere."

When the empty state is related to only parts of the content, you should use the default variant.

Empty states that are displayed on the entire page should use the large variant.

The content can be start aligned if it fits the layout better. However, on mobile the vertical layout will be used to make better use of the space.

Use a heading level that fits the context. If the empty state is the main content of the page, use h1. If it is part of a section, use h2 or lower.
