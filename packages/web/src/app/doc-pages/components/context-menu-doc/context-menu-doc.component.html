The trigger is often a 'more menu' icon button. You get a light grey color as a background when you hover. Structures as a list of buttons where the entire area is clickable.
The context menu includes responsive functionality and is suited for all devices. However you should keep in mind that it can be overwhelming on mobile if the context menu has a lot of content. The context menu can be positioned above and below the clickable trigger and aligned in the center, left or right. Use icons to help users easily recognize known actions. Different actions can be separated by headings and provide more context to the actions. Always one alternative selected, works likes a radio button. Buttons and links inside the context menu may be disabled. If you use icons, make sure to set their color to disabled yourself.
Space or Enter to open or close the popover.
ArrowUp/ArrowLeft or ArrowDown/ArrowRight to cycle between the options.
Esc to close the popover when open.
To make the context menu accessible for screen readers, remember to add the following attributes to your DOM elements:
  • aria-haspopup="menu" should be added to the trigger element.
  • role="menuitem" should be added to each menu item, if it's a regular context menu.
  • role="menuitemradio" should be added to each menu item, if it's a selectable context menu.
role="menu" is added to the overlay by default. If you are unsure of how to apply the attributes, see the example code at the top of the page.
When the trigger is selected remember to add the e-btn---selected class for the trigger.
