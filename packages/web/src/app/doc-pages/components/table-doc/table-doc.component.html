Accordion
Accordion is used when you need to have additional information to the row. An accordion (20x20px) is placed all the way to the right with the possibility to expand/collapse on the whole row. Keep the header title above empty. You can have multiple rows open at the same time (the user decides when to close/open).

  • Icons:  e-icon--expand_circle-color & e-icon--expand_circle-filled-color
Alignment
The first column should always be left aligned no matter the content of that column. By default all content is aligned to the left. There are however helper classes available for aligning content to the center or to the right. The table header should always follow the alignment of the column. Besides the first column, any type of content can be center aligned or right aligned.
Button
Use icon buttons when you need actions in your table.
Checkbox
Use checkboxes when you need to multiselect rows to perform actions. The first checkbox, in table header, should check and uncheck all checkboxes that are in the same column. Read about nested checkboxes and the indeterminate state in the checkbox documentation . Add the class e-table__checkbox--selected or e-table-mobile__checkbox-selected to all selected rows.
Footnote
Use a footnote when you need to clarify information. Use an asterix (*) to connect the data to the description.
Input
Use an input field when the user can edit values in the table.
Long column
If the table requires scrolling the max-width for a column is 450px, the class e-table__cell--multiline can then be added to ensure width of column and text wrap. Use e-text-right to align numbers to the right. The class e-table__cell--multiline-sm can be used when the column should have a max-width of 250px.
Sticky header
Sticky headers should primarily be used when there is a lot of information to scroll through, so that the user can view the context at all times when scrolling. Optional sticky headers for both horizontal and vertical scroll. You can choose to have either row headers sticky, column headers sticky or both.
Row header
The first column will be sticky on the left side including the linked column header.
Column header
The column header will be sticky on top while scrolling vertically.
 When the row headers are sticky these headers should also be sticky on mobile. The row currently displaying will follow on top until the next row is reached. Column headers will not be sticky on mobile.
No stripes
Use the modifier class e-table--no-stripes to remove the grey colored stripes on the table body.
Numbers
Use the class e-text-mono on the table cell, when comparing numbers in the table. Mono has equal width for each character, making comparing numbers easier than with Red Hat Text or Red Hat Display.
Select
Use select when navigating forward from the selected row.
Sorting
Sorting is another way to ease user's search process in tables. The sorting functionality can either be in a sorting dropdown outside of the table, or as a table sorting in the table header. Here we provide the classes for table sorting, but the logic needs to be added.
  • Figma
  • Icons:  e-icon--sorting-bold & e-icon--sorting_ascending-bold & e-icon--sorting_descending-bold
On desktop, the sorting arrow appears when hovering over any of the sortable top tabs. When clicked, the table will sort according to the chosen column; ascending (up), descending (down), or original order (no arrow).
On mobile, the function is displayed in a sorting menu, which also impacts the highlighted title.
  • Aria label: Remember to add aria-label to the table to provide a description for the table.
  • Scope: The scope attribute should be places on all table header elements to make it clear if the headers is a row or column header. Add scope="row" and the th element to the first cell on a row if this element acts as a header.
