Accordion in a table is used when you need to have additional information to the row. An accordion (20x20px) is placed all the way to the right with the possibility to expand/collapse on the whole row. Keep the header title above empty. You can have multiple rows open at the same time (the user decides when to close/open).

Use icon button when you need actions in your table. The default and recommended size for icon is 20px (with a 48 x 48px hover area). The first checkbox, in table header, should check and uncheck all checkboxes that are in the same column. Read about nested checkboxes and the indeterminate state in the checkbox documentation . Add the class e-table__select--selected or e-table-mobile__select-selected to all selected elements. The first column should always be left aligned no matter the content of that column. By default all content is aligned to the left. There are however helper classes available for aligning content to the center or to the right. The table header should alway follow the alignment of the column. If the table requires scrolling the max-width for a column is 450px, the class e-table__cell--multiline can then be added to ensure width of column and text wrap. Use e-text-right to align numbers to the right. If you include a footnote to your table, it should be positioned at the bottom outside the table using the class e-table__footnote . When the first column represents headers for the first row the column should be 'th', not 'td', and scoped with row:
<th scope="row"> January 2020 </th>
