Use icon buttons when you need actions in your table. Use checkboxes when you need to multiselect rows to perform actions. The first checkbox, in table header, should check and uncheck all checkboxes that are in the same column. Read about nested checkboxes and the indeterminate state in the checkbox documentation . Add the class e-table__checkbox--selected or e-table-mobile__checkbox-selected to all selected rows. Use an input field when the user can edit values in the table. Use a footnote when you need to clarify information. Use an asterix (*) to connect the data to the description. Use select when navigating forward from the selected row. Sorting is another way to ease user's search process in tables. The sorting functionality can either be in a sorting dropdown outside of the table, or as a table sorting in the table header. Here we provide the classes for table sorting, but the logic needs to be added.

On desktop, the sorting arrow appears when hovering over any of the sortable top tabs. When clicked, the table will sort according to the chosen column; ascending (up), descending (down), or original order (no arrow).
On mobile, the function is displayed in a sorting menu, which also impacts the highlighted title.
The first column should always be left aligned no matter the content of that column. By default all content is aligned to the left. There are however helper classes available for aligning content to the center or to the right. The table header should alway follow the alignment of the column. Besides the first column, any type of content can be center aligned or right aligned. If the table requires scrolling the max-width for a column is 450px, the class e-table__cell--multiline can then be added to ensure width of column and text wrap. Use e-text-right to align numbers to the right. The class e-table__cell--multiline-sm can be used when the column should have a max-width of 250px. Here are some accessibility rules to be aware of when setting up tables. When the first cells for each row in a table can be seen as headers for the rows, the column cell should be 'th' elements and not 'td' elements. Example:
<th scope="row"> January 2020 </th>
