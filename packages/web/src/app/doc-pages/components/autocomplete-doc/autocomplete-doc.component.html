There are two different sizes of the autocomplete; medium and small. Much like small text fields, the small autocomplete is a version of the text field that’s primarily designed for internal systems, and shouldn’t be mixed with the other input style. You can disable the basic filtering in the autocomplete by setting the hasBuiltInFilter prop to false . Then, implement your custom filtering logic.

Here's what you can to do:

  1. React to changes. Use the valueOnChange event to create a function that takes the current value as a parameter and returns a new set of matching items.
  2. Filter and sort the items. Filter and sort the generated items using your preferred methods.
  3. Update the autocomplete component. Update the autocomplete component's items prop with your filtered and sorted items to apply your custom filtering and sorting to the autocomplete suggestions.

Remember

The suggestions are intended to help the user in finding a value, but it is not mandatory for the value to be selected from that list.

Use the label in the items array to display an alternative string rather than the actual value as the visible option. The label serves solely for display purposes. When an option is selected, it is the value that will be sent back.
Use the keys to navigate through the autocomplete suggestions.
Press Enter or TAB to select a autocomplete suggestion when it is highlighted.
Press the Esc key to close the autocomplete suggestion popup.
  • aria-label: Use the ariaLabel prop to give the autocomplete an accessible name if no explicit label is provided.
