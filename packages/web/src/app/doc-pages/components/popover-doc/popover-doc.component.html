In informative popover we offer popover as a shell with options like close button, heading and text. Then you can add the content you need within the area - like text and inputs for example. Popover closes after clicking outside the popover, “X” or ESC KEY.
The heading should be short and clear. The heading is optional. You can have different type of content like text, buttons, input etc. The content will have 100% width, unless the content only consists of text, then the max-width is 450px. The trigger can be any kind of button. The popover will close if you click outside, ESC key, or the button connected to it. The close button is optional and can be removed in special cases like in a menu list.
Recommended width for text popover are 450px. This is measured with the line length and number of characters - to get a good readability. Insert input, selection controls, buttons etc. A list of actions in a popover that often are used together with a more-menu icon button. This can be used when the space is limited or when you want to group actions. Popover closes immediately after selecting action or clicking outside.
The trigger is often a 'more menu' icon button You get a light grey color as a background when you hover Structures as a list of buttons where the entire area is clickable
Use icons to help users easily recognize known actions. Different actions can be separated by headings and provide more context to the actions. Always one alternative selected, works likes a radio button. Similar actions can be grouped by a divider. Use the prop hasDivider and group the actions that should have divider between them with the class ewc-popover__list-group . The popover include responsive functionality and is suited for all devices. However you should keep in mind that it can be overwhelming on mobile if the popover has a lot of content. Popover can be positioned above and below the clickable trigger and aligned in the center, left or right.
Space or Enter to open or close the popover.
Esc to close the popover when open.
