Popover {{ description }}
A popover is a shell with options like close button, heading and text. Content like text and inputs are added inside. Popover closes after clicking outside the popover, “X” or ESC KEY.
The heading should be short and clear. The heading is optional. You can have different type of content like text, buttons, input etc. The content will have 100% width, unless the content only consists of text, then the recommended max-width is 450px. The trigger can be any kind of button. The popover will close if you click outside, ESC key, or the button connected to it. The close button is optional and can be removed in special cases like in a menu list.
Recommended width for text popover is 450px. This is measured with the line length and number of characters - to get a good readability. Insert input, selection controls, buttons etc. The popover include responsive functionality and is suited for all devices. However you should keep in mind that it can be overwhelming on mobile if the popover has a lot of content. Popover can be positioned above and below the clickable trigger and aligned in the center, left or right.
Space or Enter to open or close the popover.
Esc to close the popover when open.
When the trigger is selected remember to add the e-btn---selected class for the trigger.
