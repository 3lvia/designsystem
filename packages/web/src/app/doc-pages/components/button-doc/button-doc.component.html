Primary buttons are action-oriented buttons used for primary actions we want the user to do. Primary are design to stand out and take the most attention of all the buttons. Primary button are often used for saving, sending, submitting, registering and so on. It can be used alone or together with secondary and tertiary. Secondary buttons are the alternative we give users to the primary action and should be used in less important actions. Secondary are often used for canceling, declining, go back and so on. It can be used alone or together to support primary or link style button.' Tertiary buttons are usually used for miscellaneous actions and may not be what the user is looking for to do right then. Tertiary are designed to be less striking than primary and secondary. Example of use is download and edit. It can be used isolated alone or paired together with primary or secondary. Danger button can be used for destructive actions with high severity. The red color warns the user, but make sure to confirm the action with a confirmation screen. Icon buttons are used for well-known and simple actions such as close, edit, delete etc. They are ideal for task based actions where the button should not attract to much attention. Icon button should be supported by a tooltip. The modifier for circled icon button can be added when circled icon buttons are used alone, and results in a different hover effect than in the normal icon button. The icon buttons all have at least 32 pixel touch dimensions to ensure that they are user-friendly and accessible to everyone across different devices. It is also important that there is enough space between icon buttons and other elements to decrease the chance of accidentally pressing an incorrect target.
The e-btn class can also be used on anchor tags to make links appear like buttons.

There are three different sizes of buttons: small, medium, and large. The choice of button size depends on how important the button is and how much attention it should have. It can also depend on the components used together with the button whether they are large or small in size.

Make sure you have a good touch target on mobile. You can extend the button in full width on the mobile to make sure that the touch target is big enough for the thumb.

Use large if the button has high priority and needs a lot of attention. When using buttons in a pair, the primary button are placed to the right and secondary to the left. This should be followed at all times.
The button label should be a action verb and tell the user what the button will do. Label in a button should not be longer than three words.
Icons can be used to the left or right of the text on a button. This should only be done when used to clarify additional information that helps the context.
An exception goes for Tertiary buttons however. In these cases, icons along with texts are often are preferred, as they clarify that the otherwise simple text is clickable.
