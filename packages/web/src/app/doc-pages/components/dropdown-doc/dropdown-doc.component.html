It's possible to display icons in the dropdown. Simply add a key value pair of icon: IconName to the dropdown options to create icons aligned on the left side of the label text.
Note that to display icons, all options need to have an icon attribute. Remember to pass icon names as camelcase, as under the hood we use the icon component to display icons within the dropdown component, click here for more information on the icon component
Add a status icon on the right side to specific options of any dropdown. Use it to quickly notify the user to pay extra attention to some options over others. Each status consists of a predefined color and icon that signals their level of importance. A status should always be accompanied by an explaination to make sure the user understands the meaning of any status. Avoid overusing status - only utilize it if necessery. Choose the correct status to secure a consistent and universal user experience.

  • Error

    Notify that something is wrong. Remember to explain why or what the user can do to resolve the problem. Example: Error, missing or invalid option.

  • Warning

    Help users to avoid error situations or notify that something undesirable may occur when choosing this option. Remember to explain the reason behind the warning and how to avoid or fix the undesirable outcome.

  • Informative

    Use to inform the user that more information is available when choosing this option, unlike other options. Do not use if relevant for all options.
To add a status to an option, simply add a key value pair of status: string to the option. The available status options are error , warning and info .
The dropdown component can ignore its default 448px maximum width and expand to fill its parent's width by using the isFullWidth prop.
Use load more items if you have too many items to load and display everything from the start - if not, consider our autocomplete . Use it in similar scenarios if multiselection is required, as this feature is not supported in autocomplete. Load more items is useful if the number of items is so great that it would negatively impact the user experience due to slow loading time - utilize it to split up the data at reasonable intervals. When using load more items, be mindful of the list sorting, displaying the most important items at the top, readily available to the user and not hidden further down the list. Making the input searchable is also recommended with a large number of items in the dropdown.
When the button is clicked, the onLoadMoreItems event is emitted. The dropdown component does not handle the loading of more items, it is up to the user to implement this functionality and update the items prop. The button is hidden by default and can be shown by setting the hasLoadMoreItemsButton prop to true . When items are loading, the prop isLoadingMoreItems should be set to true . This will show a spinner in the button. When there are no more items to load, the button should be removed.
