It's possible to display icons in the dropdown. Simply add a key value pair of icon: innerHTML to the dropdown options to create icons aligned on the left side of the label text.
Note that to display icons, all options need to have an icon attribute. The icon that you pass will be parsed as innerHTML , and icons should be 24px wide. This corresponds to e-icon--sm if you use icons from elvis.
The dropdown also supports nested dropdowns. This is done by utilizing the children property on a list item. The list contains new dropdown items, so the tree can be as deep as you need it to be. Be aware that this can affect responsiveness, as nested dropdowns will occupy a lot of screen space if the tree is too deep. Note that all values should be unique, since the output only is the select value.
Add a status icon on the right side to specific options of any dropdown. Use it to quickly notify the user to pay extra attention to some options over others. Each status consists of a predefined color and icon that signals their level of importance. A status should always be accompanied by an explaination to make sure the user understands the meaning of any status. Avoid overusing status - only utilize it if necessery. Choose the correct status to secure a consistent and universal user experience.

  • Error

    Notify that something is wrong. Remember to explain why or what the user can do to resolve the problem. Example: Error, missing or invalid option.

  • Warning

    Help users to avoid error situations or notify that something undesirable may occur when choosing this option. Remember to explain the reason behind the warning and how to avoid or fix the undesirable outcome.

  • Informative

    Use to inform the user that more information is available when choosing this option, unlike other options. Do not use if relevant for all options.
To add a status to an option, simply add a key value pair of status: string to the option. The available status options are error , warning and info .
The dropdown component can ignore its default 448px maximum width and expand to fill its parent's width by using the isFullWidth prop.
Use the load more button if you have too many options to load and display everything from the start - if not, consider our autocomplete . Also, make use of it in similar scenarios if multiselection is needed, as this feature is not supported in autocomplete. The load more button is useful if the number of items is so large that it would negatively impact the user experience due to slow loading time - utilize it to split up the data at reasonable intervals. When using the load more button, be mindful of the list sorting, displaying the most important options at the top, readily available to the user, and not hidden further down the list. Making the input searchable is also recommended with a large number of options.
When the button is clicked, the onLoadMoreItems event is emitted. The dropdown component does not handle the loading of more options, it is up to the user to implement this functionality and update the items prop. The button is hidden by default and can be shown by setting the hasLoadMoreItemsButton prop to true . When options are loading, the prop isLoadingMoreItems should be set to true . This will show a spinner in the button. When there are no more options to load, the button should be removed.
