Use the e-inlined-field--full-width class if the inputs are inlined on the same line, should take up all width available.

Use the e-inlined-field class outside the form-field element to align inputs on the same line. When the zip code is filled in, the post office should be filled in automatically. The readonly attribute makes the input immutable, meaning the user can not edit the input or its value. Use readonly solely when the user cannot interact with the field, but the value is selected. Also, ensure to only use readonly if there are other interactive input elements in the form, filter, or similar.

Consider that if the input is a stand-alone element, use regular typography instead. If the value is immutable and is not selected/should not be read by the computer, use the disabled attribute instead.

The e-input---loading class should be added to indicate that the text field is loading, for instance if the current value is being validated. A span with the class e-input__suffix can be used to add a suffix to the input field. This can be used to add a unit or similar to the input field. There are two different sizes of the input component; medium and small. The small text fields are typically used in Elvia's internal systems, where screen real estate is an important factor. Regardless of which style you use for text fields, you should always stick with the same type throughout the page and never mix the two. By default, all fields must be required. Remember to add attributes to the input-fields to the describe the current state of the input to inform assistive technologies. For example required , disabled , or invalid .

During a invalid state, ensure to set aria-invalid to true and set the aria-errormessage attribute to the id of the element containing the error message text for that input. When aria-errormessage is used, the element it refers to must be visible in order for users to see or hear the error message.
