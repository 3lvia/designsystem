Checkbox {{ description }}
This example is not interactable and logic for adding the indeterminate state should be applied as described below. Use the indeterminate state, e-checkbox--indeterminate , when you have sublist of checkboxes, where only some of them are checked. This class has to be applied using logic. Make sure to always add a label that is linked to the checkbox, to make it clear for assistive tools what the checkbox is doing. Use the e-form-field classes to get correct styling of labels. The states have associated classes that can be used to programmatically add the desired styling. Remember to always add the state attribute ( checked , required , disabled , invalid and so on) to the input to inform assistive tools of the current state. Remember to add validation classes to help users understand what needs to be corrected. Read more about how the validation should happen .
