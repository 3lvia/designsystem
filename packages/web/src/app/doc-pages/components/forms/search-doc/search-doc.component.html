We have two types of search: instant and on submit. These are typically used in connection to a data table or list. Instant search is typically used when filtering among content, while search on submit is for typical search functions (when content isn’t displayed before searching). For use cases were the search result should appear "instant".
Use the e-search--searched class when the search input field has a valid search input.
For use cases where the search result should appear "on submit". Use a large button for medium search, and a small button for small. You can find button documentation here .
Use the e-search--searched class when the search input field has a valid search input.
When a search field needs to take up the entire available width, give the field the e-form-field—width-full class. Include a placeholder to suggest to users what they can search for, but try to keep it short. You don’t need to show label in search, but add an aria-label as an explicit invisable label to the search form code that can accommodate screen reader users.
