Breadcrumb {{ description }}
The breadcrumb should start with home (landing page) and end with the current page. All pages besides the current page are a link. The breadcrumb is hidden until you have two levels (no breadcrumbs are visible on the homepage).
On mobile you only get the closest parent.
 Breadcrumb is a great way to reveal the user's location and makes it easier to navigate and find what they are looking for. But it's not necessary to use breadcrumbs for single-level pages with no more than one parent category. Backlink can then be used to navigate the user back to home. To use routing with the breadcrumb component, the callback function onLinkClick can be used. Instead of passing in a URL to the breadcrumbs prop, the onLinkClick callback function is provided with the index of the clicked breadcrumb. This functionality can be used to handle routing outside of the breadcrumb component allowing the use of routing modules (e.g. Angular RoutingModule or React Router).
