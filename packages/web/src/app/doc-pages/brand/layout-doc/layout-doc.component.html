Our grid-system is based off of Bootstraps grid-system and works almost the same. Read the Bootstrap documentation . All the Bootstrap grid-system classes are available for use. To use the grid-system add the e-grid class on a wrapper element. The bootstrap-classes will only work inside this wrapper, and will therefore not break any other potential grid-systems.
Use col-sm , col-md and col-lg to change grid at breakpoints.
A grid layout for external and internal systems. If you don’t need to use all 12 columns individually, you can group the columns to create wider columns. Use justify-content , align-content , and offset to align col -classes inside grid. Use order to choose the order of columns. The grid can have levels.
Use the class e-grid-gutters-ext (for external systems) or e-grid-gutters-int (for internal systems) to add dynamic gutters on rows. Get vertical gutters as well by adding the class e-grid-gutters-vertical .
Combine the padding classes (e.g. px-lg-3 ) and negative margin classes (e.g. mx-lg-n3 ) to add your own custom gutters. Use no-gutters to remove gutters on the row.
Use the class e-grid-margin-ext (for external systems) or e-grid-margin-int (for internal systems) to add dynamic margin on rows.
Spacing refers to space between UI elements. It is important to create an appropriate spatial relationship between items and are a big part of the visual design. We follow an 8px spacing system meaning we also provide classes like e-p-8 and e-m-8 . View all classes available in the utilities table Avstand viser til mellomrommet mellom brukergrensesnitts-elementer. Det er viktig å skape et passende romlig forhold mellom elementene og er en stor del av den visuelle layouten. Vi følger et 8px spacing system som betyr at vi også tilbyr klasser som e-p-8 and e-m-8 . Se på alle de tilgjengelige klassene i utilities tabellen .
Something to keep in mind when choosing spacing: Noe å huske på når du velger avstand:
  • It's about logic. Related things are closer than unrelated things, so two unrelated things must be far away from each other. Det handler om logikk. Relaterte ting er nærmere enn urelaterte ting, så to urelaterte ting må være langt unna hverandre.
  • The bigger the object, the bigger space around it. Jo større objekt, desto større plass rundt det.
  • We all need to breathe, so let the layout breathe as well. Vi trenger alle å puste, så la også oppsettet puste.
We have defined a guide to help you choose the correct spacing. These are not rules so exceptions may occur. Spacing will also be used differently depending on which device you are on. Vi har laget en veiledning for å hjelpe deg med å velge riktig avstand. Dette er ikke regler, så unntak kan forekomme. Avstand vil også brukes forskjellig avhengig av hvilken enhet du bruker.
For spacing between related elements or inside a component use spacing between 8px and 32px.

Example:

  • Padding inside card, table, form etc.
  • Margin between typography, related input-fields, filter options etc.
 For avstand mellom relaterte elementer eller inne i en komponent, bruk avstand mellom 8px og 32px.

Eksempel:

  • Utfylling inne i kort, tabell, skjema osv.
  • Margin mellom typografi, relaterte inndatafelt, filteralternativer osv.
 For spacing between unrelated elements and defines the layout use spacing between 40px and 72px.

Example:

  • Margin between cards, tables etc.
 For avstand mellom ikke-relaterte elementer og definerer layout, bruk avstand mellom 40 px og 72 px.

Eksempel:

  • Margin mellom kort, tabeller osv.
 For spacing between different sections use spacing between 80px and 112px. This is used by external websites more often.

Example:

  • Margin between different sections vertically
 Bruk mellomrom mellom 80px og 112px for avstand mellom ulike seksjoner . Dette brukes oftere av eksterne nettsteder.

Eksempel:

  • Margin mellom ulike seksjoner vertikalt.

Boxes are used as a background container to group content and together with grid and space you can make different layouts. Boxes have no max width or height. Bokser brukes som en bakgrunnsbeholder for å gruppere innhold, og sammen med rutenett og mellomrom kan du lage forskjellige oppsett. Boksen har ingen maksimal bredde eller høyde. Besides a simple box, we have types with title outside and with a colored line. The box with a colored line should be used if you want the box to take more attention, but not use multiple boxes with colored lines on the same page. Foruten en enkel boks har vi typer med tittel utenfor og med en farget linje. Boksen med farget linje bør brukes hvis du vil at boksen skal ta mer oppmerksomhet, men ikke bruke flere bokser med fargede linjer på samme side.
