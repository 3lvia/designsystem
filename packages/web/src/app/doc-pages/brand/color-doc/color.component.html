Color tokens are labels that represent specific colors in a design system. They make it easier to maintain consistency in the use of colors across different design elements and provide a shared language for colors among designers and developers. By using color tokens, designers and developers can easily change the colors used in the design without having to hunt through the code for every instance of a particular color. This saves time and helps prevent mistakes.

All tokens are based on a hex color The base tokens refer hex color and build the basis of the design system. Use purpose tokens in all applications. These always refer to a base token. Purpose tokens have descriptive names that explain their intended use.

Tokens adapt to light and dark themes automatically, keeping their purpose token names and roles the same while adjusting their values. For instance, e-color-background-hover-1 can be a light or dark shade of green depending on the theme used.

Here are some tips and examples of how to think when chosing the tokens.
  • Disabled element: The application needs to vizualize that an elemtent is disabled (not interactable). If the element has a background with text on top, both the background and text should probably change color. In the token list we find the "Background" and "Text" categories and then the "States" sections. In this section, we find two different disabled options. The original color of the element was pretty dark and, therefore, we choose to use option 2 for both the background and text ( background-disabled-2 and text-disabled-2 ).
  • Green and red status: The application has an element which switches between green and red to show the status of something. In the token categories, we find the "Signal" category and here we can find the tokens signal-positive and signal-danger .
  • Dark background in light theme: Use the color-background-3 class to get dark surfaces within the light theme, such as in the header, footer, and "Elvia curve" elements. This class inverts the appearance of all components within the element, ensuring the content maintains a visually correct look.
  • Elvia green: The Elvia green color is used a lot of places throughout the Elvia systems, so when replacing green remember to differentiate between the use cases:
    • e-color-*-[state] for e.g. hover and selected (Text, Background and Border)
    • e-color-signal-positive for signalling something positive or successful (Signal)
    • e-color-brand-accent for logos or Elvia concept elements (Assorted)
