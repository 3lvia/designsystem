Tables are available in two levels of density. When using tables, make sure to always use the same type of density and style on all tables throughout the page. Less compact tables should be used in most external systems when displaying information that should be perceived as easy to read and provide a better experience for new users. The text in the last column will always be aligned to the right on phones. Sorting is another way to ease user's search process in tables. The sorting functionality could either be in a sorting dropdown outside of the table, or as a table sorting in the table header. Here we provide the classes for table sorting, but the logic needs to be added.

The sorting arrow appears when hovering over any of the sortable top tabs. When clicked, the table will sort according to the chosen column; either ascending (arrow up), descending (arrow down), or original order (no arrow). When active, the arrow will stay visible in the table header until it’s either deactivated or replaced by another column sorting.

Add the e-table__sort-header to the th-tag on the columns you want to sort. Then add the 2 icons for arrow up and down inside the span with class e-table__sort-icon . When toggling between arrow down and up, add or remove the class e-table__sort-icon--up . When toggling showing the arrow when active add or remove the class e-table__sort-icon--inactive .

The first column should always be left aligned no matter the content of that column. By default all content is aligned to the left. There are however helper classes available for aligning content to the center or to the right. The table header should alway follow the alignment of the column. If the table requires scrolling the max-width for a column is 450px, the class e-table__cell--multiline can then be added to ensure width of column and text wrap. Use e-text-right to align numbers to the right. If you include a footnote to your table, it should be positioned at the bottom outside the table using the class e-table-footnote .
